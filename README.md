Данный проект выполнен в ходе обучения на курсе Andersen QA Trainee.

 Что внутри:
* Тестовая документация: Чек-листы и тест-кейсы (см. папку /documentation).
  Чек-листы включают себя поверки формы логина: https://qa-course-01.andersenlab.com/login
  Тест-кейсы включают в себя проверки формы регистрации: https://qa-course-01.andersenlab.com/registration
* Баг-репорты: Примеры оформления дефектов в Jira (см. папку /bug-reports).
  Приведены примеры баг репортов по трем разным проектам (в том числе мобильного приложения)
* API Тестирование: E2E коллекция запросов (см. папку /postman).
  Создана коллекция с запросами для проверки формы регистрации, логина, изменения данных, удаления профиля, а также E2E папка запросов с написанием разных скриптов и тестов.

 Инструменты и навыки:
* REST API: Тестирование методов GET, POST, PUT, DELETE через Postman.
* Документация: Написание и актуализация тест-кейсов.
* Баг-трекинг: Работа в Jira, контроль жизненного цикла дефекта.
* SQL: Проверка данных в БД после выполнения запросов, работа с запросами

 Как запустить API тесты:
1. Импортируйте файлы из папки /postman в свой Postman.
2. Выберите окружение (Environment) и запустите коллекцию.
